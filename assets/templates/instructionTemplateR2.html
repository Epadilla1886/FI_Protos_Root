Coding Instructions

After Coding Details

  1. Create a feature branch on GitHub
    1. Navigate to your {{ repoName }} repo
    2. Click on the Branch: master button
      • Important! If you are on a branch other than master look down the list and select master to switch to the master branch
      • For the {{ repoName }} repo, all new branches will be made from the master branch
    3. Enter the branch name: {{ branch }} into the input field
    4. Click on the Create branch: {{ branch }} button
  2. Save files to GitHUb - Add all modified & created files
    1. While still in your {{ repoName }} repo on the GitHub website:
    2. Verify that you are now on the {{ branch }} branch. Look at the branch button and verify that it says Branch: {{ branch }}
    3. If you are on the correct branch, locate the {{ lfzTitle[1] }} folder in the list of folders
    4. Click on the appropriate folder for this prototype. Verify you are now in the correct folder, which should be something like:
      {{repoName + "/" + lfzTitle[1] + "/"}}
    5. If on the correct branch and in the correct folder click on the Upload Files button
    6. Drag and drop all the files required for this prototype, or select them by clicking where it says Choose your files. You will see a list of all the files that you have added at the bottom of the drag and drop area
  3. Package (group) all saved git files with a commit message
    1. Add a commit message to your files in the following format
      • "{{ lfzTitle[1] }} - Scott Bowler"
        • Scott Bowler should be replaced with your First & Last name
        • Grouping changed files with a message allows other developers to easily understand what changes you made. Commit messages typically reflect what changed in a particular version of the code.
    2. Triple check you are on the {{ branch }} branch
    3. Click the Commit changes button
  4. Make a request to have your code reviewed (Pull Request)

      Step 4.1 Open your prototypes repo in the browser

      • The link should follow this format: https://github.com/[your-user-name]/{{ repoName }}

      Step 4.2 Click thePull requests tab to open the pull request list

      Step 4.3 Click the New pull request button

      Step 4.4 Compare Changes

      In this step, there are a hand full of highlighted things that need to be taken into consideration

      1. base fork: Learning-Fuze/{{ repoName }}
        1. For all prototypes this will always remain the same
      2. base: r + month + year + _ + first initial + last name
        1. Here you will select the option that matches YOUR name
        2. If you can not locate an option that has your first initial and last name, contact an instructor
      3. head fork: [your_user_name]/{{ repoName }}
        1. Here you will select the option that matches your Github username / {{ repoName }}
        2. If you can not locate an option that has your username / {{ repoName }}, contact an instructor
      4. compare: {{ branch }}
        1. Here you will locate the branch name you created at the beginning of the prototype
        2. This will usually be the same as the name of the prototype

      Step 4.5 Add details to the pull request

      1. Give the pull request a title, by default the last commit message is pre-populated there for you
        1. If you set the commit message above, you should get {{ lfzTitle[1] }} - [your first & last name] already pre-populated in the Title Field
        2. If there is a different title pre-populated, change the title to match the correct format:
          1. {{ lfzTitle[1] }} - [your first & last name]
      2. Don't assign a user. (An Instructor will select who this get assigned too)
      3. Click on the Create pull request button
  5. Collaborate on feedback
    • Through Github.com, another developer (or in this case an instructor) will review the Pull Request you made - Does your code work? Collaboration happens through the Github UI & notifications are sent through email when when comments are added to pull requests. Be sure to check the email you used to sign up for your Github account to know when you've recieved code feedback
      • Students should make modifications to their code based on the instructors feedback
      • Instuctors will be giving an initial assessment based on the pull request. Any additional collaboration after the instructor's initial feedback will be on a requested basis only.
